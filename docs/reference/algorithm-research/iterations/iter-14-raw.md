# Iter 14 — Conformal abstention over the fused retriever score

Goal: turn the iter-5/11/12/13 + iter-8 mixture into a calibrated decider — emit a prediction set with formal coverage over `accepted_tool`, and use set size to route to **return / clarify / escalate-to-agent**. Closes the iter-13 residual gap (no honest "I don't know").

## Papers

1. **Algorithmic Learning in a Random World** — Vovk, Gammerman, Shafer (2005, 2nd ed. 2022). The foundation: nonconformity scores + exchangeability give finite-sample, distribution-free coverage. Takeaway for Mesh: we get the guarantee from a *single quantile of held-out scores* — no model retraining, fits a 13-iter pipeline as a thin wrapper. [alrw.net](https://www.alrw.net/)
2. **A Gentle Introduction to Conformal Prediction and Distribution-Free Uncertainty Quantification** — Angelopoulos & Bates (2021). The practitioner reference: split-conformal recipes, score design, classification/regression/ranking variants, code. Takeaway: with as few as ~100 calibration points you get marginal coverage in expectation; *conditional* coverage and tight sets need 500–1k+. Sets the floor for Mesh's bootstrap regime. [arxiv.org/abs/2107.07511](https://arxiv.org/abs/2107.07511)
3. **SelectiveNet: A Deep Neural Network with an Integrated Reject Option** — Geifman & El-Yaniv (2019). Pre-conformal baseline: jointly learn a classifier + selector under a coverage constraint. Takeaway: shows abstention as first-class objective, but the guarantee is *empirical* not distribution-free — motivates why conformal is the principled upgrade for Mesh. [arxiv.org/abs/1901.09192](https://arxiv.org/abs/1901.09192)
4. **Conformal Ranked Retrieval (Two-stage Conformal Risk Control)** — Wang et al. (2024). Applies conformal risk control to retrieve→rank pipelines (MSLR-WEB, Yahoo LTRC, MS MARCO), with custom retrieval-risk and ranking-risk losses. Takeaway: this is the closest published template for Mesh — gives a formal coverage knob over "accepted item is in top-k retrieved set" *without* assuming anything about the ranker. [arxiv.org/abs/2404.17769](https://arxiv.org/abs/2404.17769)
5. **Mitigating LLM Hallucinations via Conformal Abstention** — Yadkori et al., DeepMind (2024). Uses self-consistency similarity among sampled LLM responses as the nonconformity score; calibrates an abstention threshold with a distribution-free upper bound on hallucination rate. Takeaway: directly applicable to iter-12 long-context Sonnet's reranked answer — abstain when sampled answers disagree, with a *provable* hallucination ceiling. Calibration sets in the paper are O(few hundred) QA pairs. [arxiv.org/abs/2405.01563](https://arxiv.org/abs/2405.01563)
6. **Adaptive Conformal Inference Under Distribution Shift** — Gibbs & Candès (2021). Online adjustment of the conformal quantile so long-run coverage is maintained even when exchangeability breaks. Takeaway: Mesh's user base and tool catalog drift weekly — ACI keeps coverage honest as the concierge corpus shifts, with one tunable step-size. [arxiv.org/abs/2106.00170](https://arxiv.org/abs/2106.00170)
7. **Multi-model Ensemble Conformal Prediction in Dynamic Environments** — Liu et al. (2024). Selects on-the-fly from multiple candidate predictors while preserving coverage; achieves strongly-adaptive regret across intervals. Takeaway: literally the Mesh setup — 4 retrieval paths (ColBERT / BetaE / GraphRAG / agent) as the candidate models; per-query best-path selection without losing the coverage guarantee. [arxiv.org/abs/2411.03678](https://arxiv.org/abs/2411.03678)
8. **A Critical Perspective on Finite Sample Conformal Prediction Theory** — (2025). Honest critique: marginal coverage holds, but *calibration-conditional* coverage degrades sharply at small n; sets become uselessly wide or coverage drops below nominal in single-shot deployment. Takeaway for Mesh: at <100 calibration pairs, expect prediction sets to span half the catalog — conformal pays off only past ~500–1k labeled (intent, accepted_tool) pairs. [arxiv.org/abs/2512.14727](https://arxiv.org/abs/2512.14727)

## Synthesis — Conformal abstention in Mesh

**Pipeline.** Calibration set = held-out concierge conversations with explicit `(intent, accepted_tool)` labels (thumbs-up + clicked-through-and-kept signal). For each calibration example, run all four retrievers + iter-12 reranker; record per-path nonconformity score `s_p(intent, tool) = 1 − fused_score(tool | intent, path)`. Aggregate via min-score (Multi-Model Ensemble CP, paper 7) so the path that's most confident drives the score for each candidate tool. Take the (1−α) quantile of the calibration max-scores → threshold τ. At inference, emit prediction set **C(intent) = {tool : min_p s_p(intent, tool) ≤ τ}**. Route by `|C|`: **|C|=1** → answer directly; **2 ≤ |C| ≤ k_max** → clarifier follow-up scoped to the disagreement axis; **|C| = 0** or **|C| > k_max** → escalate to iter-8 agent. Wrap τ in Adaptive CI (paper 6) so the threshold drifts with the live concierge stream.

**New depth.** First *formal* statement Mesh can make to founders: "the tool you'd accept is in this set with probability ≥90%." Replaces the iter-13 learned gate, which had no coverage guarantee. Set size becomes a principled OOD detector — wide sets mean *the calibration distribution doesn't cover this query*, which is exactly when iter-8's agent earns its compute.

**Noise absorbed.** Distribution shift from new tools / new founder cohorts → ACI re-tunes τ online. Per-path miscalibration (BetaE overconfident on rare predicates, ColBERT overconfident on lexical overlap) → multi-model CP picks the locally-best score. No retraining required across iter-5/11/12/13.

**Gap that REMAINS.** (a) **Bootstrap.** Mesh starts with <100 labeled (intent, tool) pairs. Per paper 8, calibration-conditional coverage at n=100 is wide and unstable; sets will be uselessly large until n≈500–1k. We need either synthetic calibration (LLM-judge labels with quality gating) or a warm-up phase where conformal *runs in shadow* (logs set sizes, doesn't gate) until n crosses ~500. (b) **Exchangeability.** Active concierge users skew the calibration distribution toward early adopters; weighted/localized CP variants are needed once cohorts diverge. (c) **Set-size doesn't equal cost.** A size-3 set of near-duplicates is cheap to clarify; a size-3 set spanning unrelated capabilities is expensive — current CP doesn't model that.

## Suggested next direction (iter-15)

**Mixture-of-retrievers ensemble with learned per-query routing weights**, trained against the conformal calibration signal. Iter-14 gives us *which path was most confident* per labeled example for free; that's exactly the supervision a routing model needs. From the unresearched candidates list this beats the others cleanly: *agent distillation* presupposes we know which queries the agent wins on (we won't until iter-14 telemetry runs); *federated personalization* is premature pre-traction; *online learning with regret* overlaps with ACI in iter-14; *multi-task instruction-tuned retriever* is a heavyweight retrain when we still have <1k labels. A learned mixture gate, supervised by conformal-confident paths, is the smallest next step that makes the |C|=1 fast-path *better*, not just more honest — directly attacking the "set too wide to act on" failure mode left by iter-14.
