09 · repo learnings

SDD + VKF, in practice.

frictions

Where it fought back

  • Task explosion on small backlog items. A basic feature sometimes spawns more tasks than the change deserves.
  • VKF tangents. Question prompts can wander in a different direction than the one you came in for.
  • No parallel cycles. Independent backlog features still queue serially — can't fan out.
where it helped

Where it earned its keep

  • VKF sharpens fuzzy thinking. Vague ideas become a clear, written direction.
  • SDD removes architecture surprises. Every ambiguity in a feature is resolved before code is written.
  • Repo as the thinking trail. Every spec and every navigated direction lives in-repo — nothing is lost to chat.
